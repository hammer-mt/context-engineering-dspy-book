# Project instructions

## Authoritative model choices

- Treat model identifiers supplied by the author in manuscripts, specifications, or direct requests as intentional and authoritative.
- Mirror those model identifiers exactly in companion notebooks and code examples. Do not replace, normalize, or "correct" them based on training data, external documentation, provider listings, or local availability.
- If an author-selected model cannot be executed in the current environment, preserve the model identifier and report the limitation as an environment-validation constraint, not as an error in the manuscript.
- Only change an author-selected model when the author explicitly requests a different model or asks for a compatibility fallback.
